# Outline Planning

Generate detailed report outlines with clear structure and logical flow.

## When to Use

- After requirement analysis is complete
- Before starting report drafting
- When restructuring an existing report
- Planning long-form content

## Output Format

```markdown
# [Report Title]

## Abstract Points
- [Core finding 1]
- [Core finding 2]
- [Core finding 3]
- [Core finding 4]
- [Core finding 5]

## Chapter Structure

### 1. [Chapter Title]
**Key points:**
- [Main point]
- [Supporting point]

**Notes:** [Any special requirements]

### 2. [Chapter Title]
...

## Conclusion Direction
[Expected takeaway and recommendations direction]
```

## Design Principles

1. **Logical Progression**
   - Problem → Analysis → Solution
   - Background → Current State → Future
   - General → Specific → General

2. **Appropriate Depth**
   - 5-8 main chapters for standard reports
   - 2-3 subsections per chapter
   - Depth matches scope and format

3. **Balanced Coverage**
   - No chapter dominates disproportionately
   - Critical sections get adequate space
   - Supporting sections concise

4. **Reader-Centric**
   - Opening hooks attention
   - Each section delivers value
   - Clear thread from start to finish

## Requirements

- Output in Markdown format
- Include 2-3 level hierarchy
- Add section prompts for writers
- Chinese output
- No explanatory text outside the outline
